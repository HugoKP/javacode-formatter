O Projeto

Programadores geralmente se esmeram em produzir layouts de seus códigos que os tornem mais fáceis de ler. Recursos como indentação, recuos, espaçamentos, etc, são essenciais para se obter uma boa legibilidade do código.

A estas técnicas de formatação se somam os recursos de colorização de código aplicados pela maioria das IDEs, que destacam classes de identificadores por meio de cores e fontes especiais.

Portanto, se um programador tiver interesse em construir um site ou página para documentar algum projeto seu, ou mesmo um pequeno trecho de código, é necessário que o código exibido pelo navegador seja diagramado segundo o padrão que já nos habituamos ao ler e manipular fontes de programas no editor de alguma IDE.

É útil também que as linhas do código estejam devidamente numeradas. Para que a documentação possa fazer referência e explicar trechos específicos do programa.

Mas como qualquer iniciante em HTML deve saber, se nós apenas "colarmos" um trecho de código fonte (de uma linguagem qualquer) em um documento HTML, toda a formatação se perde. Além do quê, evidentemente, este código não seria também exibido com os recursos de colorização costumeiramente aplicados pelos editores das IDEs modernas.

Este projeto visa desenvolver um programa em Java para auxiliar na criação de documentação HTML de códigos fonte também em linguagem Java.

O objetivo é que o programa possa receber como entrada um arquivo texto com um fonte Java e produzir um outro arquivo texto HTML capaz de exibir este mesmo referido fonte Java devidamente diagramado em qualquer navegador onde o documento for aberto.

Ele deverá ser mostrado como as mesmas indentações e layout com que foi escrito e, além disso, no arquivo HTML gerado as linhas de código estarão numeradas e identificadores de elementos como classes, métodos, constantes, etc, e também strings e caracteres literais, serão estilizados com cores e padrões que auxiliem no seu reconhecimento visual.
